Criando e Dando Permissão a um Usuário
Início  Anterior  Próximo

No desenvolvimento de aplicações geralmente há um equipe de desenvolvedores envolvida e nessa equipe os desenvolvedores podem haver papéis específicos. O WI_Builder oferece a possibilidade de adiconar usuários que posssam acessá-lo e a cada um desses usuários pode ser associado um papel específico para cada um dos projetos.

Para criarmos um usuário pelo WI_Builder clique no link Administração da barra de menus e clique na opção Usuários, identificada pelo ícone . Como exemplo vamos criar um usuário cujo nome de login seja tutorial, para isso no campo Nome para login digite o login do novo usuário que neste caso será tutorial, no campo Senha entre com uma senha que será usada para validação do login de entrada para o WI_Builder e no campo Descrição entramos com alguma descrição para esse usuário que pode ser o seu nome completo e em seguida clique no botão "gravar".

Se no momento da criação do usuário a opção Master for marcada, automaticamente será dada permissão para que esse usuário tenha acesso completo e irrestrito a todos os projetos e telas de configuração do ambiente do WI_Builder. Para traçar um paralelo, um usuário com privilégio master seria o mesmo que um usuário com privilégio root em ambientes UNIX. O usuário admin é exemplo de um usuário com privilégio master e que sempre estará disponível em qualquer ambiente WI_Builder.

Agora que criamos um usuário vamos dar permissão para que ele possa acessar o projeto tutorial pelo WI_Builder. Escolha a opção Permissões que é identificada pelo ícone . A tela de definição de permissões é composta de três colunas, a coluna da esquerda tem o campo Usuários Cadastrados que lista todos os usuários cadastrados no WI_Builder e logo abaixo a combo Grupo que lista os tipos de grupo (papel) a que um usuário pode pertencer, a coluna do meio tem os botões que indicam quais ações podem ser feitas sobre os usuários e a coluna da direita tem o campo Usuários desse projeto que lista o(s) usuário(s) que tem permissão para acessar o projeto atualmente aberto pelo WI_Builder.
